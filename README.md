# sport-ued-hub

## 项目简介

sport-ued-hub 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为前端页面资源的集中存储与版本管理空间。

项目主要面向需要长期维护、可独立访问的静态页面集合，适合作为前端开发、设计原型或小工具页面的发布载体。

## 目录说明

```
sport-ued-hub/
├── pages/          # 存放所有独立 HTML 页面，每个页面为一个独立文件或子目录
├── assets/         # 公共资源文件（CSS、JS、图片等），按需组织
├── docs/           # 项目文档或说明性文件（可选）
└── README.md       # 本文件
```

- `pages/` 是核心目录，每个 HTML 页面可独立放置在根目录下或按功能分组至子目录中。
- `assets/` 用于存放跨页面共享的样式、脚本或媒体资源，避免重复引入。
- `docs/` 可存放设计稿、使用说明或变更日志等附加文档。

## 页面归档说明

本仓库中的每个 HTML 页面均为独立可运行的静态页面，不依赖后端服务或特定域名。页面之间互不耦合，可单独通过 `file://` 协议或 GitHub Pages 等方式访问。

归档时请遵循以下约定：

- 页面文件名使用小写字母、数字和连字符，如 `example-page.html`。
- 若页面依赖外部资源，请尽量使用相对路径引用 `assets/` 目录下的文件。
- 每个页面应包含基本的 HTML 结构（`<!DOCTYPE html>`、`<head>`、`<body>` 等），便于直接预览。
- 如有多个页面属于同一主题，可将其放入 `pages/` 下的子目录中，并在子目录内添加 `index.html` 作为入口。
